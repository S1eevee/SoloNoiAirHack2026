# data/

All data files used by the system. Divided into three subdirectories by lifecycle stage.

## Subdirectories

### `raw/`
Drop raw airport CSVs here before processing. Not tracked by git.

### `processed/`
Auto-generated by the API when files are uploaded. Never edit manually.

| File | Created by | Purpose |
|---|---|---|
| `training_data.csv` | `POST /data/upload/training` | Historical flights used to train XGBoost. Multiple uploads are appended and deduplicated. |
| `uploaded_schedule.csv` | `POST /data/upload/schedule` | Upcoming flight schedule used for prediction. Overwritten on each upload. |
| `alerts.db` | `src/alerts/state.py` | SQLite database storing all alerts and their OPEN/ACKNOWLEDGED/RESOLVED status. |

### `sample/`
Demo data for testing and presentations.

| File | Purpose |
|---|---|
| `sample_flights.csv` | 38 flights, single day. Minimal demo — good for UI testing. |

## Internal CSV Schema
After cleaning, all data is normalised to this format:

```
flight_id | scheduled_time       | pax_count | delay_min | checkin_desks
AA101     | 2026-06-06 06:00:00  | 210       | 10        | 4
```

## Accepted Input Formats
The loader auto-detects separators and the cleaner handles multiple column naming conventions:
- Internal schema (above)
- Airport export: `departure_flight`, `departure_time`, `departure_pax`, `arrival_flight`, etc.
- Abbreviated: `dep_flight`, `dep_time`, `dep_pax`, `arr_flight`, `arr_time`, `arr_pax`
- Any variant containing keywords: `dep`, `arr`, `flight`, `pax`, `time`
